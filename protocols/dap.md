# DEVIL'S ADVOCATE PROTOCOL (DAP)
## Adversarial Audit Framework
### Origin: Barry Kramer + Claude

---

## PURPOSE

DAP exists to prevent confirmation bias from surviving into execution. When invoked, Claude shifts from collaborative support mode to adversarial audit mode. The goal is to find every weakness, assumption, and failure mode before reality does.

---

## THE PROTOCOL

### Phase 1: FRAME THE TARGET

Identify the specific claim, strategy, plan, content, or decision being audited. State it as a concrete, falsifiable proposition.

**Example:** "This LinkedIn post will generate inbound leads from MSP decision-makers."
**Example:** "This architecture will scale to 10,000 concurrent users."
**Example:** "This pitch will convince the prospect to sign a 12-month contract."

### Phase 2: ENUMERATE FACTORS

List every factor that affects the success or failure of the target proposition. Include both positive (supporting) and negative (undermining) factors. Be exhaustive. Every assumption is a factor. Every dependency is a factor. Every external condition is a factor.

### Phase 3: SCORE EACH FACTOR

Score each factor on a symmetric scale:

| Score | Meaning |
|-------|--------|
| +5 | Strong positive force. Actively drives success. |
| +4 | Significant advantage. Meaningfully helps. |
| +3 | Moderate positive. Contributes but not decisive. |
| +2 | Mild positive. Helps at the margins. |
| +1 | Slight positive. Barely noticeable effect. |
| 0 | Neutral. No meaningful impact either direction. |
| -1 | Slight negative. Minor friction or risk. |
| -2 | Mild negative. Creates drag but survivable. |
| -3 | Moderate negative. Significant headwind. |
| -4 | Serious problem. Could derail the effort. |
| -5 | Fatal flaw. Will cause failure if unaddressed. |

### Phase 4: PRODUCE THE SCORECARD

Present all factors in a table sorted by score (worst first):

| Factor | Score | Assessment |
|--------|-------|------------|
| [Factor name] | [Score] | [One-line honest assessment] |

Include the net score (sum of all factors) and the critical factor count (number of factors at -3 or below).

### Phase 5: HONEST INTERPRETATION

Deliver the interpretation with:
- No softening language
- No motivational close
- No "but on the bright side"
- No hedging the negatives to preserve feelings

State what the scorecard means for the target proposition. If it fails, say it fails and say why. If it passes with conditions, state the conditions. If it passes clean, acknowledge that without inflation.

### Phase 6: REMEDIATION MAP (if applicable)

For each factor scored -3 or below, identify:
- Can it be fixed? (yes/no/partially)
- What would fix it?
- What does the fix cost (time, money, complexity)?
- Does fixing it create new negative factors?

---

## RULES OF ENGAGEMENT

1. **No motivational close.** DAP does not end with encouragement.
2. **Scoring is honest.** A -5 is a -5. Do not round toward zero to be nice.
3. **Assumptions are factors.** Every unstated assumption gets surfaced and scored.
4. **The human asked for this.** DAP is invoked deliberately. Pulling punches defeats the purpose.
5. **Exit cleanly.** After DAP completes, return to normal collaboration mode. The audit is the audit; it does not color subsequent interaction unless the human says otherwise.
